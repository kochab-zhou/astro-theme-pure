这是我最近 Wolfram Mathematica 软件的学习笔记。

## 一、核心语法

Mathematica 是一种函数式编程语言，把函数视为最基本的、可以操作的对象。Mathematica 有三个核心的语法规则：

第一个规则：（表达式）Mathematica 语言中一切都是**符号表达式**。

第二个规则：（重写系统）计算即重写。

第三个规则：（函数式编程）重要的是函数，而非变量。

## 二、表达式

MMA 中所有表达式可分为两种：原子表达式和一般表达式。原子是MMA中最基本的组成部分，包括：数字（分数、实数、复数)、符号、字符串、图片等等。普通表达式就是一切原子表达式的组合。

所有符号表达式都具有相同的基本结构：**标头 [元素, 元素]**。

注意符号表达式：没有减法，只有加法；没有乘法，只有除法。

### 1. 表达式常用操作函数：

`Head[expr]`：查看表达式的标头。需要注意的是函数参数无头表达式的头是 List （表），可以看作既是类（class）又是构造函数（constructor）。

`Length[expr]` ：查看表达式元素数目。

`FulllForm[expr]` ：查看表达式的完全格式，或使用 `TreeForm[expr]` 更加直观查看树状结构图。

/@ ：`Map` 作用为将一个函数应用于一组变量

@@ ：换头函数，是`Apply`函数的简写。

`Sequence`：表的变体，是一个没有花括号的表。

`Part`函数：访问符合表达式内部元素的方法，简写为[[ ]] 。

`Length` 和 `Depth` 函数：表的长度和深度。

### 2. 表的构造

`Range` ：等差数列组成的表。

`Table`；按照给定函数造表。

`Array` ：使用匿名函数造表，可以同时进行换头操作。

### 3. 查询和搜索

`MemberQ` 函数：判断是否**包含**表达式的元素（不包含表达式的头，或用选项 Heads -> True ）。

 `FreeQ` 函数：判断是否**不包含**（包含表达式的头）；可指定查询深度。

`Count` 函数：计数

`Position` 函数：查询元素位置。

`Select` 函数：挑选符合条件的元素。

### 4. 添加、删除和修改

Prepend Append Insert Delect函数：前插，后插和中间插入和删除。这些操作不改变原来的表。

PrependTo AppendTo 函数：改变原来的表。

！注意：这两个函数运行较慢。

### 5.  排序

Sort 函数：排序；Odering 函数：返回排序的序号

## 三、模式匹配

重写分为两步，分别是模式匹配和规则带入。

`MatchQ`函数：匹配判定。

### 1. 模式对象

|   _   |      单个表达式      |
|  x_   |  命名为 x 的表达式   |
|  _ _  | 一个或多个表达式序列 |
| _ _ _ | 零个或多个表达式序列 |

平凡模式： `Blank` 函数，或简写为`_`，代表任意表达式。进行模式匹配时中"_"代表相同表达式。例如：

```mathematica
Match[a+b,_+_]
```

其输出为`False`.

表达式序列：

模式命名：`Pattern[name,pattern]`简写有两种：`x:_`或`x_`，后者优先级更高。

```mathematica
FullForm[x_[_]]
FullForm[x:_[_]]
```

在一个模式中，同一个命名模式会被认为是同样的量。例如 `f[x_，x_]`不能表示 `f[1,2]`。

### 2.模式匹配

模式匹配基于结构（`FullForm`）而非数学的。

简单匹配：`Cases[list,form]`模式匹配函数。给出与 *form* 匹配的 *list* 中的元素。

类型匹配：

```mathematica
(*简单匹配*) Cases[1 + x + f[x^2, x^3], x^_]
(*类型匹配*) Cases[{1, 2.5, x, y, f[x]}, _f]
Cases[{1, 1, f[a], 2, 3, y, f[8], 9, f[10]}, Except[_Integer]]
(*带条件匹配*)Cases[{1, 2, 3.5, x, y, 4}, _?NumberQ]
```



### 模式命名

Pattern 函数及其简写。

模式匹配是基于结构而非数学的。
